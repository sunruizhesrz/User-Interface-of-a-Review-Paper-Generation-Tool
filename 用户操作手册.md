**更新日期**：2026 年 4 月

## 一、系统概述

综述论文自动生成工具是一款基于 **LangGraph + RAG** 技术栈的学术写作辅助系统。用户只需输入研究主题，系统将自动完成论文检索、文本解析、知识入库、大纲规划和综述撰写全流程，最终输出结构完整的学术综述文档，支持 Markdown / PDF / Word 三种格式下载。

**当前版本（Sprint 1）**为界面演示版本，核心功能以 Mock 模式运行，后续 Sprint 将逐步接入真实后端模块。

## 二、环境准备

### 2.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS 12+、Ubuntu 20.04+ |
| Python 版本 | Python 3.10 及以上 |
| 内存 | 建议 8GB 以上 |
| 网络 | 需要访问 arXiv / Semantic Scholar（部分功能需翻墙） |

### 2.2 安装依赖

**第一步：创建虚拟环境（推荐）**

```bash
# 使用 venv
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows PowerShell
```

**第二步：安装依赖包**

```bash
pip install gradio>=4.0
pip install python-docx>=1.1
pip install reportlab>=4.0

# 可选：用于 PDF 导出（优先方案）
# 下载并安装 pandoc：https://pandoc.org/installing.html
```

**第三步：确认安装成功**

```bash
python -c "import gradio; print(gradio.__version__)"
# 应输出 4.x.x
```

## 三、快速启动

### 3.1 启动界面

在项目根目录执行：

```bash
python app.py
```

启动成功后，终端会显示：

```
Running on local URL:  http://localhost:7860
```

用浏览器访问 `http://localhost:7860` 即可打开 Web 界面。

### 3.2 命令行模式（pipeline.py）

如果不需要图形界面，可以直接通过命令行运行主流程：

```bash
# 基础用法
python pipeline.py "大型语言模型"

# 完整参数
python pipeline.py "大型语言模型" --year 2022 --papers 15 --lang 中文 --output my_survey.md
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `topic`（必填） | 研究主题 | — |
| `--year` | 检索起始年份 | 2020 |
| `--papers` | 最大文献数量 | 10 |
| `--lang` | 综述语言（中文 / English / 中英双语） | 中文 |
| `--output` | 输出文件路径 | `survey_output.md` |

```
├─────────────────────┴──────────────────────────────────────┤
│  快速示例（点击可自动填入示例参数）                              │
└────────────────────────────────────────────────────────────┘
```

### 4.2 输入参数说明

**① 研究主题（必填）**

在文本框中输入您想生成综述的研究领域，建议使用具体的学术词汇以获得更好的检索效果。

- 推荐输入示例：
  - `大型语言模型在自然语言处理中的应用`
  - `Retrieval-Augmented Generation for Knowledge-Intensive NLP`
  - `图神经网络在分子生物学中的应用`

**② 起始年份**

通过滑块选择文献检索的时间起点，范围为 2010—2025 年，默认值为 2020 年。

- 建议：关注最新进展请选 2022 年以后；需要完整技术演进背景请选 2018 年以前。

**③ 最大文献数量**

设置系统检索并处理的论文数量上限（5—50 篇），默认值为 20 篇。

- 数量越多，综述覆盖面越广，但处理时间也会相应增加。

**④ 综述语言**

从下拉菜单选择输出综述的语言格式：

| 选项 | 说明 |
|------|------|
| 中文 | 全部内容以中文撰写，适合中文期刊投稿 |
| English | 英文综述，适合国际期刊 |
| 中英双语 | 同时输出中文和英文版本 |

### 4.3 开始生成

填写完参数后，点击 **「🚀 开始生成综述」** 按钮，系统将按以下流程自动执行：

| 步骤 | 操作 | 预计耗时 |
|------|------|---------|
| Step 1 | 检索论文元数据（arXiv / Semantic Scholar） | 5—30 秒 |
| Step 2 | 下载 PDF 全文 | 1—5 分钟 |
| Step 3 | 解析 PDF，提取文本块 | 30 秒—2 分钟 |
| Step 4 | 向量化，写入知识库 | 1—3 分钟 |
| Step 5 | 规划综述大纲 | 10—30 秒 |
| Step 6 | 逐章生成综述内容 | 2—10 分钟 |
| Step 7 | 校验引用，整合输出 | 30 秒 |

> **注意**：Sprint 1 版本为 Mock 模式，以上步骤均为模拟执行，总耗时约 5 秒。

执行过程中，**运行日志框**将实时显示每步进度，顶部进度条同步更新。

### 4.4 查看综述结果

生成完成后，综述内容将以 **Markdown 格式**在右侧预览区渲染展示，包含自动生成的各章节内容。

### 4.5 下载导出

结果区域底部提供三个下载按钮：

| 按钮 | 格式 | 适用场景 |
|------|------|---------|
| ⬇ 下载 Markdown | `.md` 文件 | 进一步编辑、导入其他工具 |
| ⬇ 下载 PDF | `.pdf` 文件 | 直接阅读、打印、投稿附件 |
| ⬇ 下载 Word | `.docx` 文件 | 在 Word 中继续编辑修改 |

> **Sprint 1 说明**：当前版本点击下载按钮将提示功能待完善，完整导出功能将在 Sprint 2 实现。

## 五、多格式导出（代码调用方式）

如需在代码中直接调用导出功能，可以使用 `exporter.py` 提供的接口：

```python
from exporter import export

# 准备综述内容（Markdown 字符串）
content = "# 大型语言模型综述\n\n## 1. 引言\n..."

# 导出为 Markdown
path_md = export(content, fmt="md", output_dir="./output", filename="survey")

# 导出为 Word
path_docx = export(content, fmt="docx", output_dir="./output", filename="survey")

# 导出为 PDF（需要 pandoc 或 reportlab）
path_pdf = export(content, fmt="pdf", output_dir="./output", filename="survey")

print(f"文件已保存：{path_md}")
```

## 六、常见问题与排错

### Q1：启动时报 `ModuleNotFoundError: No module named 'gradio'`

**解决方案**：运行 `pip install gradio>=4.0` 安装依赖。

### Q2：界面显示 `[Warning] graph.py 未找到，将使用 Mock 入口函数`

**说明**：这是正常的 Sprint 1 运行状态，系统将自动切换到 Mock 模式，不影响界面演示。待成员 A 的 `graph.py` 就绪后，将此文件放置到同一目录下即可自动切换为真实模式。

### Q3：点击"开始生成"后没有反应

**排查步骤**：
1. 检查研究主题是否填写（不能为空）
2. 查看终端是否有报错信息
3. 尝试刷新浏览器重新访问 `http://localhost:7860`

### Q4：PDF 导出失败，提示 `pandoc 不可用`

**解决方案**：安装 reportlab（`pip install reportlab`）或安装 pandoc（访问 [pandoc.org](https://pandoc.org/installing.html) 下载对应系统安装包）。

### Q5：Word 导出失败，提示 `python-docx 未安装`

**解决方案**：运行 `pip install python-docx>=1.1`。

### Q6：如何修改默认端口？

在 `app.py` 末尾找到 `demo.launch()` 调用，修改 `server_port` 参数：

```python
demo.launch(server_name="0.0.0.0", server_port=8080)
```
